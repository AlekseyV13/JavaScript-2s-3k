## 👤 Інформація про виконавця
* **Студент:** Войчук Олексій

У ході роботи над модулем замовлень (Orders) було успішно реалізовано та протестовано повний життєвий цикл бізнес-логіки, а також усунено серію критичних архітектурних багів:

Реалізація ACID-транзакцій: Налаштовано логіку створення замовлень через QueryRunner. Тепер при купівлі товару його кількість на складі (stock) автоматично перевіряється та зменшується. У разі будь-якої помилки відбувається повний rollback бази даних.

Усунення багу циклічної рекурсії (Stack Overflow): Виправлено конфлікт eager: true зв'язків у TypeORM, який призводив до нескінченного циклу при генерації JSON та неочікуваного падіння сервера. Методи findAll, findOne та updateStatus переведено на повернення очищених, "пласких" об'єктів.

Броньована валідація користувача з JWT: Створено надійний парсер getCleanUserId для вилучення ID користувача з токена авторизації. Це повністю вирішило проблему з брудним синтаксисом JSON-рядків та помилками типу invalid input syntax for type integer у Postgres.

Тестування безпеки та RoleGuard: Успішно перевірено роботу обмеження прав доступу. Зміна статусу замовлення через ендпоінт PATCH надійно блокує запити від звичайних клієнтів (403 Forbidden) та відпрацьовує зі статусом 200 OK лише після ескалації прав користувача до admin.

Результат: Сервіс повністю працездатний, покриває всі бізнес-вимоги, успішно протестований через PowerShell та стабільно працює всередині Docker-контейнерів.
